# Security Policy

## Public-demo boundary

Do not submit real customer, student, patient, government, employer, export-controlled, classified, or regulated data to this repository or its public demo. Do not commit credentials, OAuth tokens, private keys, or production endpoints.

## Reporting a vulnerability

Please report suspected vulnerabilities privately to the repository owner rather than opening a public issue that contains exploit details or secrets.

## Supported version

The latest default branch is the only supported public-demo version.

## Required production changes

Before using this pattern with real data, add enterprise identity, durable approval storage, secret management, network policy, dependency locking/SBOM, centralized telemetry, penetration testing, privacy review, and an authorization process appropriate to the environment.
